General Settings
Global bot behaviour for MOTD rotation.
🕐 Daily Fire Time
The bot sends the MOTD once per day at this time. Changing this takes effect on the next scheduling cycle — no restart needed if the new time is still in the future today.
📊 Priority Order
The bot checks these in order and stops at the first match:
1. Specific date — exact MM-DD match in the Specific Dates tab. Always wins.
2. Date range — first matching range from the Date Ranges tab. Blends with Default by weight.
3. Weekend — Saturday & Sunday, if no range matched. Blends with Default by weight.
4. Default pool — fallback for all other days.
Default Pool
Used on regular weekdays when no range or specific date applies. Also blends into themed pools via weight.
⚖️ Weight
1.0
During themed periods, the chance of using this pool is: default_weight / (themed_weight + default_weight). A weight of 1.0 with a themed pool at 0.7 means ~59% chance of getting a default message even during that theme.
💬 Messages
Weekend Pool
Used automatically on Saturdays and Sundays when no date range matched first. No range entry needed.
⚖️ Weight
1.0
💬 Messages
Date Ranges
Activate a named pool for a period. First matching range wins. Use MM-DD for yearly recurring, YYYY-MM-DD for one-off dates like Easter.
📅 Configured Ranges
ℹ️ Date format guide
MM-DD e.g. 10-15 — repeats every year. Supports new-year wrap (e.g. 12-20 → 01-06).
YYYY-MM-DD e.g. 2026-04-02 — fires only in that specific year. Update annually for moving holidays like Easter.
Specific Dates
Override with an exact message on a specific day every year. Highest priority — beats everything else.
📌 Date overrides
Days of Week
Map a day of the week to a pool. Checked after date ranges, before falling back to default. Only days listed here are affected.
📆 Day → Pool mapping
ℹ️ How days work
Map any day to any pool name. To replicate the old weekend behaviour, add saturday → weekend and sunday → weekend.
You can point multiple days at the same pool (e.g. both saturday and sunday → weekend), or give each day its own unique pool.
If the named pool doesn't exist or has no messages, the bot falls through to the default pool automatically.
